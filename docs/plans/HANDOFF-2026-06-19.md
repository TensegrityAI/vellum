- **`TextBuffer`** (`crates/core/src/buffer.rs`): now backed by `ropey::Rope` (ADR-0006). Byte-offset

- **`event`** (`event.rs`): `EditEvent::{Inserted,Deleted}` (`#[non_exhaustive]`), `apply`,

  Selection always clamped to a **char boundary** after every edit (the H2a Critical fix).
- **`cursor`** (`cursor.rs`): `Selection { anchor, head }` — grapheme/word movement, `byte_range()`
  (ordered), pure value object (still unit-tested; also used internally by `Document`).

## Deferred debt & threaded notes (THE AUDIT SHOULD PROBE THESE FIRST)

Everything below was consciously postponed or noted during build/review. The deep audit should
verify each is genuinely benign-for-now (and not hiding a bug), and decide what to remediate.

**Architecture / model**
- **Unbounded undo/redo history.** `Document` holds `undo`/`redo: Vec<EditEvent>` with no cap /
  snapshot / compaction. ADR-0002 accepts this for Inc 1; a memory-pressure vector under scripted
  bulk edits. Consider a history cap when the view wires real input.
- **Inverse-stacks ≠ replayable event log.** `Document` mutates the rope in place + keeps inverse
  stacks; it is *not* literal "state = fold(events)". Fine for Inc 1 undo/redo, but time-travel/CRDT
  (future) need a real append-only log. Noted in ADR-0002 Consequences.
- **`text() -> String` O(n) materialization** on every `Document::delete` (slice), every
  grapheme/word nav, and every wasm `tokens()`. `TODO(inc1+)` proposes `TextBuffer::slice(range)
  -> Cow<str>` over rope chunks. Throws away rope locality on each edit; fine at prompt sizes.
- **`ByteRange` is a named-field struct** while the other newtypes are tuple structs — deliberate
  (reads better), cosmetic.

**Editor semantics**
- **Type-over-selection = two undos.** `insert_at_cursor` over a non-empty selection records a
  `Deleted` then an `Inserted` (two history entries), so it takes two Ctrl+Z to fully revert. Most
  editors coalesce to one. Documented on `insert_at_cursor` + ADR-0008; coalescing deferred
  (`TODO(inc1+)`).
- **Word-jump "every-edge" convention.** `prev/next_word_boundary` stop on *every* segment edge,
  including whitespace runs (so `move_word_right` on "foo bar" hits byte 3 AND 4). Most editors skip
  whitespace. The primitive is the right low-level foundation; the editor-feel "skip whitespace"
  policy belongs one layer up — **apply it in I5 when wiring Ctrl/Alt+Arrow keybindings.**
- **Selection not auto-rebased after external edits.** Inc-1 contract (pinned by a test): core does
  NOT shift a caret to track edits, except its own edit methods set+clamp the caret. The **view (I5)
  owns rebasing** of any selection it holds across edits.

**Language / tokenizer**
- **Range-scoped tokenize can split a `{{ }}` block at a non-aligned range edge** → different edge
  tokenization vs whole-doc. Accepted for Inc 1 (the view passes block-aligned / whole-doc ranges;
  wasm always passes `0..len`, which is byte-identical to legacy). True incremental re-lex = Inc 2.
- **`Severity` has no `Ord`.** Variants are declared in descending severity; deriving `PartialOrd,
  Ord` later is non-breaking. Inc-2 diagnostics sorting will want it. (G1 review M-1.)

**Testing / CI**
- **Property tests deferred to Inc 2** (plan). The test audit flagged three invariants worth a
  `proptest` to de-risk: offset round-trip (`utf16_to_byte∘byte_to_utf16 == id`), inverse involution,
  undo/redo idempotence. Currently covered by hand-picked cases only. **Consider pulling at least
  the offset round-trip proptest forward** — it's the blocker-1 contract everything sits on.
- **wasm-bindgen tests don't run under `cargo test`** (they show as "0 tests"); only `wasm-pack test
  --node crates/wasm` runs them (11). **CI MUST run `wasm-pack test`** or the entire JS-boundary
  trap-safety suite is ungated. Verify `.gitlab-ci.yml` does.
- **Standing CI optimization** (from the 2026-06-15 handoff, still open): a prebuilt CI image
  (rust 1.89 + node + bun + wasm-pack + cargo-deny) to kill cold-install time.
