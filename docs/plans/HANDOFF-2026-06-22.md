## ➡️ NEXT: Task I2 — diff-based mutation (closes Inc-1 blocker #1)

**Goal:** replace `view.ts`'s `syncCoreToValue` full clear+reinsert (O(n)/keystroke,
and only char-boundary-safe because it always deletes the *whole* buffer) with a
minimal diff converted UTF-16→byte, so each edit touches only the changed range.

**Steps (TDD where pure):**
1. **`computeDiff(oldValue, newValue) -> { utf16Start, utf16RemovedLen, inserted }`**
   — a pure helper in `ts/view/src/` (e.g. `diff.ts`). Find the common prefix and
   common suffix in **UTF-16 code units**, with astral/multibyte care (do not split
   a surrogate pair — clamp the prefix/suffix boundaries off mid-surrogate). TDD it
   hard (`diff.test.ts`): ASCII insert/delete/replace, emoji, CJK, combining marks,
   no-op (equal strings), full replace, insert-at-0, append-at-end.
2. **Wire it in `mountVellum`'s `onChange`:** compute the diff vs the last known
   value, convert `utf16Start`/`utf16Start+utf16RemovedLen` to **byte** offsets via
   the wasm helpers `editor.utf16_to_byte(...)` (already exposed in H2b,
   **non-trapping** → returns `Result`/throws a JS `Error` on a bad offset — handle
   it), then `editor.delete(byteStart, byteEnd)` for the removed run and
   `editor.insert(byteStart, inserted)` for the new text. **Use the wasm helpers,
   NOT a hand-rolled converter** (audit + plan are explicit).
3. Keep a `lastValue` the view holds to diff against; update it after each apply
   and after programmatic pushes (`input.setValue` on undo/redo, later).
4. Remove `syncCoreToValue`; keep `render()` painting via the Highlight API.

**Acceptance:** typing multibyte text (emoji/CJK) round-trips through the core with
no trap and only the changed range mutated; existing vitest stays green; demo types
live. (Note: full DOM/visual acceptance is via chrome-devtools MCP, now connected.)

**Then:** I3 per-instance highlight namespacing (`vellum-${id}-keyword` etc.; scope
`clearHighlights` per instance — two editors must not clobber the global
`CSS.highlights`); I4 `MeasurePort` + arithmetic layout + virtualization; I5
caret/selection render + keyboard→core movers + **composition (widens the
`InputSource` port)** + apply the whitespace-skip word-jump policy. Phase J:
